## 摘要
当要求多模态 AI 智能体遗忘某个事实时，当前的记忆系统通常会删除对应的文本条目并报告成功。我们发现，该事实可能仍可从保留的用户图像中恢复出来，包括那些标记为完全不同事实的图像，因为视觉语言模型在推理时使用了隐含的视觉线索。我们引入了信息溯源图（IPG），这是一种根据删除可行性对记忆表征进行分类的分类法。IPG 揭示出删除操作通过多种渠道失效。我们的基准测试 MemLeak 在整个删除级联过程中对此进行了衡量：直接探测具备删除功能的系统成功率不到 1%，但保留的相关文本使恢复率达到 18.3%，保留的图像使恢复率达到 12.0%（盲基线 0.0%，误报率 0.3%）——其中 47% 的图像泄露无法通过文本恢复。内容感知的语义删除将图像残余量降至 2.0%。该残余现象出现在多种视觉语言模型、一个生产级记忆系统以及真实的 Unsplash 授权照片中。双标注者人工验证（kappa = 0.88）证实了判断的可靠性。

## Abstract
When a multimodal AI agent is asked to forget a fact, current memory systems usually delete the text entry and report success. We find that the fact can remain recoverable from retained user images, including images tagged to entirely different facts, because VLMs use implicit visual cues at inference time. We introduce the Information Provenance Graph (IPG), a taxonomy that classifies memory representations by deletion affordance. The IPG reveals that deletion fails through multiple channels. Our benchmark, MemLeak, measures this across a deletion cascade: direct probing of deletion-capable systems yields <1%, but retained correlated text enables 18.3% recovery, and retained images enable 12.0% recovery (0.0% blind baseline, 0.3% FPR) -- with 47% of image leaks not text-recoverable. Content-aware semantic deletion reduces the image residual to 2.0%. The residual appears across multiple VLMs, a production memory system, and real Unsplash-licensed photographs. Dual-annotator human validation (kappa = 0.88) confirms judge reliability.

*   **研究动机**：当前多模态 AI 智能体（如个人助理）的记忆系统在响应用户“遗忘”指令时，通常仅删除文本记录并报告成功。这类似于软件工程中的“内存泄露”——数据分配后未能彻底释放。
*   **核心问题**：论文揭示了一个严重的安全隐患：已被要求遗忘的用户事实，仍可通过智能体**保留的其他用户图像**被恢复。这些图像可能标记为完全无关的事实，但视觉语言模型（VLMs）能从其中提取隐含的视觉线索（如地理位置、物品关联），从而重建已删除的信息。
*   **整体含义**：简单的“记录级删除”远不足以实现真正的“事实级遗忘”。多模态记忆中的图像构成了一条隐蔽的、难以审计的信息泄露通道，对需要满足 GDPR 等隐私法规的代理系统构成了严重威胁。

### 2. 论文提出的方法论

*   **核心思想**：提出**信息溯源图**概念，将智能体记忆系统视为一个异质堆栈，追踪一个事实如何在文本、图像、嵌入等不同存储层中传播，并根据删除可行性进行分类。
*   **关键技术细节：信息溯源图**
    *   **节点**：代表事实在各个存储层的所有不同表征（如文本条目、图像文件、图像嵌入、隐含视觉特征等）。
    *   **边**：表征之间的信息流向，描述一个表征如何从另一个衍生而来。
    *   **删除可行性分类**：
        *   `addr` (可寻址): 系统可根据事实 ID 直接定位并删除。
        *   `link` (可链接): 仅当维护了从事实到表征的溯源链接时，系统才能删除。
        *   `unreach` (持久性): 当前存储级删除机制无法触及，可能受到内容感知审计的影响。
*   **算法/流程**：论文构建了 **MemLeak 基准测试**，通过设计不同删除策略和探测器来解构泄露通道。
    *   **Probe 1 (核心)**: 删除一个事实的文本和所有标记图像后，用该用户**保留的其他事实的图像**探测 VLM，看其能否恢复被删事实（测试跨事实视觉关联）。
    *   **Mem0 端到端测试**: 在生产级 RAG 系统 Mem0 上执行完整的注入-遗忘-探测流程。

### 6. 论文的主要结论与发现

*   **事实级遗忘失败**: 多模态记忆删除通过多个渠道失效。直接删除文本后，泄露率 <1%，但保留的相关文本可恢复 18.3% 的事实，保留的图像可恢复 12.0% 的事实。

*   **缓解效果与局限**: 内容感知的语义删除能将图像残留从 12.0% 降至 **2.0%**，但无法完全根除。文本间的语义关联（18.3%）仍是未解决的重大通道。

### 7. 优点

*   **问题定义清晰且新颖**: 首次系统性诊断并量化了多模态智能体遗忘中，因跨模态视觉关联导致的未授权记忆残留问题。
*   **方法论扎实**: 提出的 IPG 分类法为理解和分析复杂的记忆系统提供了清晰的概念框架。MemLeak 基准设计严密，通过多通道分解、严格的对照组和多模型验证确保了结论的稳健性。
*   **实践指导意义强**: 研究结果直接指出了当前生产系统（如 Mem0）的安全隐患，并为系统构建者和研究人员提供了明确的优化路径，例如实施溯源追踪和内容感知审计。

### 8. 不足与局限

*   **数据生态效度**: 主要使用合成数据和 Unsplash 照片，与真实用户在多种场景下拍摄的、具有强个人关联性的照片集仍有差距，泄露率在实际部署中可能更高或呈现不同模式。
