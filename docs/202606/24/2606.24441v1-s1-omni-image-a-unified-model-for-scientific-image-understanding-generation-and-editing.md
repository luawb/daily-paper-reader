## 摘要
我们提出S1-Omni-Image，一个开放权重的统一多模态模型，用于科学图像的理解、生成与编辑。与通用图像生成模型不同，科学图像任务不仅需要高保真合成，还需要对科学语义、结构关系、领域知识和任务意图的深入理解。为此，S1-Omni-Image基于科学多模态推理骨干S1-VL-32B，并将其理解能力与图像生成模块耦合，采用统一的先思考后生成范式。给定用户指令，模型首先生成面向任务的推理轨迹、文本答案以及任务特殊标记；它们的隐藏状态随后注入生成模块，以条件化图像生成或编辑。S1-Omni-Image在统一框架内支持科学图像理解、生成和编辑。在生成方面，它专注于科学插图和文本渲染，包括逻辑图、关系对比图、数据图表和逼真的科学可视化。在编辑方面，它将分割和其他领域特定视觉任务转化为原生图像编辑问题，支持多轮插图编辑、医学和地理图像分割、医学图像转换以及科学图像超分辨率。我们构建了包含314K训练样本的SciGenEdit数据集，并发布了模型权重、推理代码和SciGenEdit-10K。实验表明，S1-Omni-Image在保持从S1-VL-32B继承的科学图像理解能力的同时，显著改进了科学图像生成和编辑。它在GenExam和TechImage-Bench上优于开源模型，在MSD、cigRockSEM、SynthRAD2025和IXI四个编辑基准上取得了最优结果，并在科学图像理解评估中保持稳定性能。

## Abstract
We present S1-Omni-Image, an open-weight unified multimodal model for scientific image understanding, generation, and editing. Unlike general-purpose image generation models, scientific image tasks require not only high-fidelity synthesis, but also robust understanding of scientific semantics, structural relations, domain knowledge, and task intent. To this end, S1-Omni-Image builds on the scientific multimodal reasoning backbone S1-VL-32B and couples its understanding capability with an image generation module under a unified think-before-generate paradigm. Given a user instruction, the model first produces a task-oriented reasoning trace, a textual answer, and a task special token; their hidden states are then injected into the generation module to condition image generation or editing. S1-Omni-Image supports scientific image understanding, generation, and editing in a unified framework. For generation, it focuses on scientific illustrations and text rendering, including logical diagrams, relational comparisons, data charts, and realistic scientific visualizations. For editing, it casts segmentation and other domain-specific vision tasks as native image editing problems, enabling multi-turn illustration editing, medical and geographic image segmentation, medical image translation, and scientific image super-resolution. We construct SciGenEdit, a 314K-sample training dataset, and release the model weights, inference code, and SciGenEdit-10K. Experiments show that S1-Omni-Image substantially improves scientific image generation and editing while preserving the scientific image understanding capability inherited from S1-VL-32B. It outperforms open-source models on GenExam and TechImage-Bench, achieves state-of-the-art results on four editing benchmarks including MSD, cigRockSEM, SynthRAD2025, and IXI, and maintains stable performance on scientific image understanding evaluations.

## 2. 方法论文核心技术细节

### 2.1 总体架构与Think-Before-Generate范式
模型输入包含文本指令及可选的输入图像，输出则根据任务可以是文本答案、推理过程、生成或编辑图像。

1. **理解与推理阶段**：利用 Stage-I 微调后的**S1-VL-32B（32B参数视觉-语言骨干）**，自回归生成包含三部分的序列：
   - **推理链（Reasoning Trace）** `r`: 任务导向的显式推理；
   - **文本答案（Textual Answer）** `a`: 对指令的文本解释或回应；
   - **任务特殊标记（Task Special Token）** `s`: `<image_gen>` 表示生成，`<image_edit>` 表示编辑。
   
2. **隐藏状态注入**：在生成上述序列的同时，**收集骨干网络产生的所有隐藏状态（Hidden States）** `H = {h₁, h₂, ..., hₙ}`，其中每个 `hᵢ ∈ R⁵¹²⁰`。这些表征融合了输入文本、图像内容、推理过程中的语义与任务意图信息。

3. **生成与编辑阶段**：
   - **推理-扩散对齐层**：一个MLP投影层 `fₐₗᵢ𝓰ₙ(H) → C ∈ R³⁵⁸⁴`，将骨干网隐藏状态映射到MMDiT生成模块的条件空间（维度3584，与Qwen-Image的Qwen2.5-VL编码器对齐）。
   - **MMDiT图像生成/编辑模块**：基于Qwen-Image的架构，使用Flow Matching范式预测速度场（而非噪声）。给定VAE编码的目标潜在变量 `z_data` 和噪声潜变量 `z_noise`，在时间步 `t` 构建插值 `z_t = t·z_data + (1-t)·z_noise`，目标速度 `v_t = z_data - z_noise`。DiT输入 `(z_t, t, C)`，输出预测速度，最小化均方误差：
     `L_FM = E[‖ v_θ(z_t, t, C) - (z_data - z_noise) ‖²₂]`
   - **任务路由**：根据 `<image_gen>` 或 `<image_edit>` 标记决定执行生成（无输入图像仅文本条件）还是编辑（输入原图+指令，输出编辑后图像）。

### 2.2 三阶段训练策略
- **Stage I: 科学推理调优**：在完整SciGenEdit数据集（约314K样本）上，仅训练骨干网的语言模型部分，监督其生成推理链、答案和任务标记。**不训练图像输出**，仅让模型学会任务理解和推理。冻住视觉编码器。（损失：自回归语言模型损失）
- **Stage II: 推理-扩散对齐预热**：在平衡语料库（约40K提示）上，**仅训练MLP对齐层**，最小化其输出 `A_φ(h_src)` 与标准Qwen2.5-VL条件编码器输出 `h_tgt` 间的MSE。骨干网和DiT均冻结。目标：将5120维的S1-VL隐藏空间映射到3584维的DiT条件空间。
- **Stage III: 联合生成与编辑训练**：使用约294K SciGenEdit生成/编辑样本，冻结骨干网与VAE，**联合训练MLP对齐层和DiT模块**。在Flow Matching损失基础上，额外加入包含MSE和余弦相似项的**对齐正则化损失**（λ=0.8），确保科学推理表征与DiT输出空间协调适应。

## 6. 主要结论与发现

1. **统一范式有效**：“先思考后生成”范式显著提高了科学图像生成的结构正确性与布局合理性，优于直接映射方式。模型在五个科学编辑基准中有四项取得最优（SOTA），并大幅领先通用模型，证明科学场景下理解注入的必要性。
2. **能力无损融合**：在增加图像生成/编辑模块后，模型在MMMU（82.7分）、MathVision（76.7分）等核心推理基准上效果接近甚至部分超越原骨干S1-VL-32B，证明了能力的良好继承，实现了真正的多模态统一。
3. **强大的开源基线**：作为Open-Weight模型，在科学插图生成上优于Qwen-Image、GLM-Image，虽仍落后于GPT-Image-2等闭源巨头，但大幅缩小了差距。
4. **多任务统一化处理**：成功将传统上独立的医学分割、模态转换（合成CT）、视觉超分辨率统一为**“指令+输入图像→推理→编辑后图像”**的标准流水线，且结果优于专用模型。

## 7. 论文优点与亮点

- **范式创新性强**：提出“**先思考后生成**”方法论，将大模型推理过程的隐藏状态作为扩散模型的强条件，这在科学图像领域属于新颖且有效的设计。
- **系统化统一方案**：用一个模型完成理解、生成、编辑三类任务，并将多种专用视觉任务统一为自然语言指令编辑，接口简洁，潜力巨大。
- **高保真结构控制**：在科学插图的逻辑关系、箭头指向、模块排版等结构性要求上，显著优于通用开源模型，对实际科研出版有直接帮助。
- **全面的科学编辑覆盖**：涵盖医学、地质、病理等多学科的特色评估基准，且均达到SOTA，证明了模型的扎实泛化力。
- **开放科学精神**：开放模型权重、推理代码及数据集子集（Apache 2.0协议），为科学多模态社区提供了高价值且可复现的基线。
- **中英双语指令支持**：演示中展示了处理中文学术插图提示的能力，实用性更广。

## 8. 不足与局限性

- **文本渲染仍是软肋**：尤其是**中文长文本及密集标注**，会出现字形错误、笔画缺失、模糊或语义替换，这在要求精确定义的科研插图中是硬伤。
- **复杂指令遵循有缺陷**：面对精细的局部结构调整或空间重排指令，模型有时“理解但未执行”，输出与原图高度一致，显式执行编辑的能力不够稳健。
- **训练策略待深化**：因资源与数据限制，未进行科学图像大规模预训练、偏好对齐（DPO/RLHF）或针对生成/编辑的强化学习，导致在美学、复杂指令跟随及文本渲染上与商业旗舰模型差距明显。
- **局限的模态与领域**：主要聚焦于2D科学图与医学影像。未涉及3D模型、化学反应式、谱图、蛋白质结构等更广泛科学模态。
- **安全与伦理风险泛化**：模型可能被滥用于伪造实验科学证据、篡改医学图像或处理敏感生物数据。论文虽警示并建议“仅作辅助生成”，但仍存在潜在的合规和科研诚信风险。
- **长序列编辑累积偏移**：多轮迭代编辑下，可能出现细节漂移，论文未提供系统性量化评估。
