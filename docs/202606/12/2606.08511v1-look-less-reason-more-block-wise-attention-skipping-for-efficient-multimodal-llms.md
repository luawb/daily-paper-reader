## 摘要
多模态大语言模型（MLLMs）因在长视觉标记序列上自注意力的平方计算成本而面临显著的推理瓶颈。然而，我们发现当前架构中存在一个关键的效率低下问题：视觉注意力饱和。我们的分析表明，视觉标记在早期层中迅速建立其空间结构和模态内关系，使得更深层中的视觉到视觉自注意力在计算上变得冗余。相反，这些层中的前馈网络（FFNs）对于将视觉特征投影到不断演化的文本语义空间中仍然至关重要。利用这一洞察，我们提出了Visual-Skip（V-Skip），一种无需训练的推理范式，该范式将空间交互与语义演化解耦。V-Skip不是丢弃标记，而是通过选择性地绕过饱和的视觉自注意力模块来施加逐块的结构化稀疏性。此外，认识到不同的下游任务需要不同的推理深度，V-Skip采用轻量级的少样本校准来动态路由任务最优的稀疏路径。大量实验表明，V-Skip有效地绕过了冗余的视觉注意力以实现逐块稀疏性，在各种MLLMs上保持了94.16%至100.31%的性能保留率。最终，我们证明，为了更有效地推理，模型不需要丢弃它们看到的东西——它们只需要在合适的深度“少看”一些。

## Abstract
Multimodal Large Language Models (MLLMs) face a significant inference bottleneck due to the quadratic computational cost of self-attention over long visual token sequences. However, we identify a critical inefficiency in current architectures: Visual Attention Saturation. Our analysis reveals that visual tokens rapidly establish their spatial structure and intra-modal relationships in early layers, rendering visual-to-visual self-attention in deeper layers computationally redundant. Conversely, Feed-Forward Networks (FFNs) in these layers remain essential for projecting visual features into the evolving textual semantic space. Leveraging this insight, we present Visual-Skip (V-Skip), a training-free inference paradigm that decouples spatial interaction from semantic evolution. Rather than discarding tokens, V-Skip imposes block-wise structured sparsity by selectively bypassing saturated visual self-attention modules. Furthermore, recognizing that varying downstream tasks demand distinct reasoning depths, V-Skip employs a lightweight, few-shot calibration to dynamically route the task-optimal sparsity path. Extensive experiments demonstrate that V-Skip effectively bypasses redundant vision attention to achieve block-wise sparsity, maintaining a 94.16% to 100.31% performance retention across diverse MLLMs. Ultimately, we prove that to reason more effectively, models do not need to discard what they see -- they simply need to "look less" at the right depth.

- **核心问题**：多模态大语言模型（MLLM）在推理时，因视觉 token 数量急剧增加（高分辨率输入可达数千个 token），导致自注意力计算呈平方复杂度，产生严重的推理延迟和内存瓶颈。现有加速方法（如 Token Pruning、Token Merging、Layer Skipping）往往采取“破坏性”手段——永久删除 token 或冻结整层，虽然降低了计算量，却破坏了视觉信息的完整性或语义演化的连续性，增加了目标幻觉风险。
- **整体含义**：论文从“视觉 token 处理的功能演化”出发，发现**视觉注意力饱和**现象——早期层负责空间结构聚合，深层视觉自注意力变得冗余，但前馈网络（FFN）对语义对齐依旧关键。基于此，作者提出一种“少看多想”的加速范式：不过度丢弃信息，而是在合适深度减少冗余的视觉自注意力计算，保留语义演化路径，实现高效且无损的推理。

### 2. 论文提出的方法论

#### 2.1 核心思想：解耦空间交互与语义演化
- 视觉 token 在深层自注意力饱和后，视觉-视觉自注意力计算冗余，但逐 token 的 FFN 变换对视觉特征投射到文本语义空间至关重要。
- **V-Skip** 通过**逐块结构化稀疏**，在深层跳过视觉 token 之间的自注意力，同时保留全部 token 的 FFN 变换，既消除了平方计算瓶颈，又维持了语义对齐。

#### 2.2 关键技术细节
- **视觉信息增益 (Visual Information Gain, VIG)**：衡量跳过某一层视觉自注意力对最终输出分布的影响。
  - 定义为：\( G_k(l) = \mathbb{E}_{X \sim D_k} [D_{KL}(P(y \mid X) \parallel P_{\neg l}(y \mid X))] \)
  - 通过 20-shot 校准集（从目标任务数据集随机采样 20 个图文对）计算每层的 VIG 值。
- **饱和层识别**：根据任务 k 下的 VIG 曲线，选取**最低的 N 层**作为饱和层，形成跳过层集合 \( \mathcal{L}^{(k)}_{\text{skip}} \)。
- **解耦注意力掩码**：
  - 视觉 token 作为 query 时：完全不与视觉 token 交互，跳过该部分自注意力计算（视觉绕过），等价于对注意力掩码做结构化稀疏。
  - 文本 token 作为 query 时：保持全注意力（可关注所有视觉和文本 token）。
- **FFN 保留**：所有 token 正常通过前馈网络，保证语义演化不中断。
- **计算复杂度降低**：标准自注意力 \( O((N_v+N_t)^2) \) → V-Skip 下降为 \( O(N_t^2 + N_t N_v) \)，去除了主导项 \( N_v^2 \)。

#### 2.3 流程概要
1. 对目标下游任务，用 20-shot 样本快速计算 VIG 曲线。
2. 选取 N 个 VIG 最低的层作为视觉注意力跳过层。
3. 推理时在这些层中实施解耦掩码，保持 FFN 完整，实现推理加速。

### 6. 论文的主要结论与发现

1. **视觉注意力饱和**是 MLLM 普遍存在的现象：深层视觉自注意力冗余，但 FFN 对语义演化必不可少。
2. V‑Skip 能够有效跳过约 60% 的饱和视觉自注意力层，在 LLaVA 系列上实现 **98.42%–100.31%** 的性能保留，在 Qwen 上达到 94.16%，显著优于 VTW 和 ShortV。
3. 保留 FFN 的解耦设计是保持语义对齐、避免幻觉的关键；V‑Skip 在 OCRBench 等精细任务上明显优于破坏性剪枝方法。
4. 任务特定的少样本校准虽然轻量，但对最大化性能保留至关重要；跨任务迁移时性能可能下降，证明了任务感知路由的必要性。
5. V‑Skip 可与 token 剪枝方法组合使用，进一步压缩计算量，甚至恢复部分剪枝带来的性能损失。

### 7. 优点（方法或实验设计的亮点）

- **非破坏性加速**：不删除、不冻结 token，完整保留视觉上下文，在减少计算的同时避免了信息丢失和特征失配。
- **精准的功能解耦**：指出注意力与 FFN 在深层功能不对称，仅针对冗余部分施加稀疏性，理论支撑清晰。
- **训练无关、即插即用**：无需重训练或微调，校准代价极低，易于部署。
- **任务自适应路由**：基于 VIG 的少样本校准动态选择跳过层，相比固定掩码更具灵活性，鲁棒性高。
- **实验充分且透明**：覆盖主流 MLLM 架构、多维度消融、等计算量对比、与剪枝方法组合，结论可信度高。

### 8. 不足与局限

- **任务依赖校准**：需要为每个下游任务单独采集少量样本进行 VIG 计算，虽然开销小，但在任务频繁切换的场景下不够“一键式”。
- **跨域泛化有限**：不同任务（如通用 VQA 与密集文字识别）的饱和层分布不同，直接复用校准结果会导致性能下降（如 OCRBench 的迁移实验）。
- **高分辨率下的性能波动**：LLaVA‑NeXT‑7B 的 MME 分数下降 3.7%，说明在极高分辨率下，深层视觉交互仍可能保留一定的重要性，方法的无损性并非绝对。
- **架构敏感性**：Qwen2.5‑VL 因注意力窗口机制的不同，需要更保守的跳过预算和重新校准，通用性有待进一步提高。
- **仅针对 Prefill 阶段**：主要加速预填充（prefill）的视觉自注意力计算，对自回归解码阶段的优化有限，整体端到端加速比例受限于视觉 token 占比。
